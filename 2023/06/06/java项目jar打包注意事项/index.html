java项目jar打包注意事项

发表于2023-06-06|更新于2023-06-09|-java -SpringBoot
|字数总计:638|阅读时长:2分钟|阅读量:

出现问题的原因

有没有小伙伴和我一样，平时很少将我们项目打成jar包，导致只是会一个大概流程很多细节都没关注，今天我就来讲讲我所遇到的坑。

1：首先我们使用的是maven打成jar

如下图我已经打成好了jar包

完成以上步骤后，进入我们的window命令窗口，这里有些小伙伴没有Open in选项，可以直接找到target文件目录在如下图所示，在按Enter键弹出cmd命令窗口。

在我们的window命令行输入，如下指令java -jar XXX架包名称,出现如下问题解决方案：

问题：Exception in thread “main” java.lang.ClassNotFoundException: com.example或者就是找不到或无法加载主类，可以使用如下方式解决。

接下来我给大家一个干净的maven-plugin插件配置

注意 标签里面的内容是，你的springBoot引导类类地址。

选中引导类右键鼠标，选择Copy Path

将地址复制到pom文件插件部分找到我们的mainClass标签中

接下来因为我们已经打包过一次本地有缓存需要先clean在package,如果还不放心缓存或者有缓存影响，可以先clean在compile,在clean最后在package。

如果遇到这样错误，不要慌：Failed to execute goal org.apache.maven.plugins:maven-clean-plugin:3.1.0:clean (default-clean) on project spring-cloud-order-service: Failed to clean project: Failed to delete

这是因为我们的cmd命令窗口占用导致我们clean失败，解决方式很简单关闭你的cmd命令窗口，在idea里的话

现在我们clean完成，继续package，接下来就和第一步开始一样找到我们的target文件下面，在命令行下打开输入java -jar 架包名。

现在看是不是没有问题啦 注意我就是被chatGPT坑了下，他给我的pom配置插件如下图，请使用我上面给的pom的maven打包插件配置。

我们的jar在我们的window环境下可以运行后才能把他放到我的Linux环境运行，中间可能也会出现许许多多问题例如我们的java环境不一致，但是不要放弃，问题是怪物解决他你才能收获经验不断升级。
